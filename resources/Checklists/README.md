# NTLM Deprecation Checklists

This folder contains comprehensive checklists to guide your NTLM deprecation and hardening journey.

## Available Checklists

### Phase-Based Checklists
- `phase1-assessment.md` - Initial NTLM usage assessment
- `phase2-planning.md` - Planning and preparation
- `phase3-hardening.md` - Implementation of NTLM restrictions
- `phase4-monitoring.md` - Ongoing monitoring and maintenance

### Role-Based Checklists
- `security-team-checklist.md` - Tasks for security operations team
- `infrastructure-team-checklist.md` - Tasks for infrastructure/AD team
- `application-team-checklist.md` - Tasks for application owners

### Specialized Checklists
- `executive-readiness.md` - Executive briefing and approval checklist
- `incident-response.md` - NTLM-related incident response procedures
- `compliance-checklist.md` - Compliance and audit requirements

## How to Use These Checklists

1. **Start with Phase 1** - Begin with the assessment checklist
2. **Track Progress** - Mark items as complete as you go
3. **Customize** - Adapt checklists to your environment
4. **Document** - Keep records of decisions and exceptions
5. **Review Regularly** - Update checklists as your environment evolves

## Checklist Format

Each checklist includes:
- Clear action items
- Prerequisites
- Estimated time/effort
- Risk level
- Dependencies
- Success criteria

## Tips for Success

- Don't skip the assessment phase
- Get executive buy-in early
- Test in dev/test environments first
- Document all exceptions and workarounds
- Plan for rollback scenarios
- Communicate with stakeholders throughout
